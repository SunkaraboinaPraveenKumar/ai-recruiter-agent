## AUTHENTICATION SYSTEM (JWT — Custom, No Third-Party Auth)

Build a complete custom authentication system with the following:

### Database Schema (users table):
- id (uuid, primary key)
- email (varchar, unique, not null)
- password_hash (varchar — bcrypt hashed)
- full_name (varchar)
- avatar_url (varchar, nullable)
- google_id (varchar, nullable — for OAuth)
- role (enum: 'recruiter' | 'admin', default 'recruiter')
- created_at, updated_at (timestamps)

### Auth Flow:
1. **Sign Up:** POST /api/auth/register — accept full_name, email, password → hash password with bcrypt (saltRounds: 12) → save user to DB → generate JWT access token (expires in 7d) + refresh token (expires in 30d) → set both as httpOnly cookies → return user object
2. **Sign In:** POST /api/auth/login — validate email/password → compare bcrypt hash → generate and set JWT tokens → return user
3. **Google OAuth:** Implement Google OAuth 2.0 flow — /api/auth/google → redirect to Google → callback at /api/auth/google/callback → create or update user with google_id → issue JWT tokens → redirect to dashboard
4. **Sign Out:** POST /api/auth/logout → clear httpOnly cookies
5. **Token Refresh:** POST /api/auth/refresh → validate refresh token → issue new access token
6. **Auth Middleware:** Create a middleware.ts that protects all /dashboard/* routes. If no valid JWT cookie found, redirect to /sign-in. Attach decoded user to request headers for downstream API routes.
7. **useAuth hook:** Create a client-side React context + hook that holds current user state, exposes login/logout/register methods, reads session from a GET /api/auth/me endpoint.

### Auth Pages:
- /sign-in — email/password form + "Continue with Google" button. Redirect to /dashboard on success.
- /sign-up — full_name, email, password, confirm password. Redirect to /dashboard on success.
- Both pages should be visually polished with the app branding, matching the overall dark/modern UI theme.

## MURF FALCON TTS INTEGRATION

Murf API endpoint: POST https://api.murf.ai/v1/speech/stream

Request body:
```json
{
  "text": "question text here",
  "voiceId": "en-US-Neural2-F",
  "model": "GEN2",
  "format": "MP3",
  "sampleRate": 24000
}
```

Headers: `api-key: MURF_API_KEY`

Handle streaming audio response:
- Use fetch with ReadableStream
- Pipe chunks to Web Audio API AudioContext
- Show speaking animation (pulsing glow on AI avatar) while streaming
- On stream end → stop animation → activate Speech Recognition

Create a server-side API route /api/murf/speak that proxies the Murf request (to keep API key server-side). Accept {text, voiceId} and stream the audio response back to client.

## GEMINI AI INTEGRATION

Use Google Generative AI SDK (@google/generative-ai):

**Functions to implement in /lib/ai/gemini.ts:**

1. `generateJobDescription(title: string): Promise<JobDescription>` — generate full job posting JSON
2. `matchCandidatesToJob(job: Job, candidates: Candidate[]): Promise<MatchResult[]>` — returns array of {candidate_id, match_score, match_reason} sorted by score
3. `parseResume(resumeText: string): Promise<CandidateProfile>` — extract structured data from resume text
4. `generateNextQuestion(context: InterviewContext): Promise<string | null>` — returns next question or null if done
5. `generateInterviewReport(transcript: TranscriptItem[], jobTitle: string, interviewType: string): Promise<InterviewReport>` — full scoring and analysis

All calls use model: "gemini-1.5-flash" for speed/cost efficiency.
